ALGORITHM FLOW (Detailed)
=========================

For each 3-dataset combination (in priority order):
  PRIORITY_TRIOS = [(3,4,5), (2,3,4), (1,2,3), ...]

  1. LOAD DATA
     ├─ Load 3 graphs (pickle files)
     ├─ Load 3 degree dictionaries (CSV)
     └─ Load seed motifs (CSV)

  2. PROCESS SEEDS
     For each seed motif (a1, b1, c1, type):
       
       2.1 FIND MATCHING MOTIFS
           ├─ Get structure of seed: (edges between a,b,c)
           ├─ Load all motifs of same type from datasets 2,3
           ├─ Filter to only matching structures
           └─ Collect matching triplets [(a2,b2,c2), (a3,b3,c3)]
       
       2.2 EXPAND FOR EACH MATCHING TRIPLET
           Seed mapping: {0:(a1,a2,a3), 1:(b1,b2,b3), 2:(c1,c2,c3)}
           
           EXPANSION LOOP (max 1000 iterations):
             a) Find all neighbors of current subgraph
             b) Get candidates with matching (in_deg, out_deg)
             c) Sort by total degree (in + out)
             d) For each candidate (n1, n2, n3):
                  IF can_add_to_mapping(n1,n2,n3):
                    → Add to mapping
                    → Break to next iteration
                  ELSE:
                    → Try next candidate
             e) If no candidates added: CONVERGED
       
       2.3 TRACK BEST RESULT
           IF len(new_mapping) > best_size:
             → Update best_result
             → Print progress
       
       2.4 SAVE CHECKPOINT
           EVERY CHECKPOINT_INTERVAL seeds:
             → Save best_result to JSON
             → Allows resume on failure

  3. OUTPUT BEST RESULT
     ├─ Write solution.csv
     ├─ Write summary.json
     └─ Print summary to stdout


ISOMORPHISM VERIFICATION (can_add_to_mapping)
==============================================

For each new candidate node triple (n1, n2, n3):

  For all existing nodes (m1, m2, m3) in current mapping:
    
    1. Check forward edge (existing → new)
       ├─ has_edge(m1, n1) in G1?
       ├─ has_edge(m2, n2) in G2?
       ├─ has_edge(m3, n3) in G3?
       └─ ALL THREE MUST MATCH (same edge/non-edge)
    
    2. Check backward edge (new → existing)
       ├─ has_edge(n1, m1) in G1?
       ├─ has_edge(n2, m2) in G2?
       ├─ has_edge(n3, m3) in G3?
       └─ ALL THREE MUST MATCH
    
    IF any mismatch → return False
    ELSE continue to next existing node

  RESULT: True if all edge relationships consistent


KEY PARAMETERS & TUNING
=======================

Speed Optimization:
  - Reduce CHECKPOINT_INTERVAL (less I/O)
  - Reduce max_iterations (fewer expansion steps)
  - Reduce candidate attempt limit

Quality Optimization:
  - Increase max_iterations (more thorough)
  - Increase max_no_progress (try harder per seed)
  - Increase candidate attempt limit

Balance:
  Default settings: ~1-2 hours for good solution
  Fast mode: ~30 minutes, smaller solutions
  Thorough mode: ~4+ hours, larger solutions

When resuming (automatic on restart):

  ✓ All results accumulated properly

Benefits:

ALGORITHM COMPLEXITY
====================

Time Complexity per seed:
  - Seed matching: O(M²) where M = motif count per type (~100-500)
  - Expansion iterations: O(I) where I ≤ 1000
  - Per iteration:
    - Find candidates: O(N) where N = graph size (20K-100K)
    - Degree filtering: O(C log C) where C = candidate count (~1000)
    - Isomorphism check: O(C * S) where S = current size (~10-100)
  - Total per seed: O(M² + I * C * S)
  - Overall: O(Seeds * (M² + I * C * S))

Space Complexity:
  - Graph storage: O(V + E) for each graph
  - Degree mapping: O(V)
  - Current mapping: O(N) where N = solution size
  - Total: O(V + E)
