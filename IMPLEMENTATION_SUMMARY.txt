Algorithm Design
================

APPROACH: Seed-Based Greedy Expansion with Isomorphism Verification

Step 1: Initialize
  - Load 5 graphs from pickle files (BANC, FAFB, MANC, MAOL, MCNS)
  - Load neuron degree info (in_degree, out_degree)
  - Load 3-node seed motifs from CSV

Step 2: Main Loop (for each 3-dataset combination)
  - Priority: MANC-MAOL-MCNS (male datasets, most homologous)
  
  For each seed motif in dataset 1:
    a) Find matching structure in datasets 2 and 3
    b) Create initial mapping (3 nodes)
    c) Iteratively expand:
       - Find all neighbors of current subgraph
       - Filter by degree sequence (must match for isomorphism)
       - Rank by total degree (in + out)
       - For each candidate triple:
         * Check if all edge relationships are consistent
         * If valid: add to mapping
         * If not: try next candidate
    d) Continue until no valid candidates remain
    e) Track best result (largest N)

Step 3: Output
  - Save best mapping to solution.csv
  - Save statistics to summary.json
  - Save checkpoint files for recovery

Key Features
============

✓ GRACEFUL FAILURE
  - Try multiple candidates before giving up
  - Catch and log all exceptions
  - Save checkpoints periodically
  - Can resume from partial results

✓ EFFICIENT SEARCH
  - Degree-based filtering reduces O(n³) to O(k³)
  - Highest-degree prioritization finds large circuits quickly
  - Early termination options available

✓ ROBUST VERIFICATION
  - Isomorphism check for every candidate node
  - Bidirectional edge verification
  - Preserves directionality

✓ COMPREHENSIVE REPORTING
  - Solution CSV with proper formatting
  - JSON summary with statistics
  - Per-dataset-trio results
  - Edge counts and neuron counts

    - Run overnight
    - Use parallel processing on multiple trios

Notes on Design
===============

Why Highest-Degree Nodes?
  - High-degree nodes have more constraints
  - Fewer candidates to check at each step
  - Tend to form stable circuit cores
  - Heuristic works well for real neural circuits

Why Seed Motif Matching?
  - Avoids requiring neuron properties (which weren't provided)
  - Pure graph-based matching
  - Flexible to different motif types (SC, BR, FL, etc.)
  - Seeds provide initial consistent structure

Why Greedy Expansion?
  - MCIS is NP-complete in general
  - Greedy approximation is practical
  - Works well with high-degree prioritization
  - Can find global optimum for many real instances

Why Checkpointing?
  - Handles large search spaces
  - Allows recovery from interruptions
  - Enables parallel processing on multiple trios
  - Useful for long-running jobs
